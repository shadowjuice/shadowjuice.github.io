第二天

今天做了几道基础题目。讲一下过程吧。
都是些misc的题目，前两道题目就不说了，看图说话，算是了解一下提交的格式吧
第三道题是那个最低有效位，上网查了查然后安装了那个stegsolve的jar包，把图片导入进去，发现跟网上说的不一样，并没有那个最低位0，最低位从2开始的，看了看发现原来是可以显示逐层的图片的，结果就发现一个二维码，扫码get到flag
第四道题exif的，这个就是安装上exiftool，然后把图片下载下来，在终端输入exiftool secret.jpg就出来这个图片的exif信息，通过浏览分析，发现其中有一行比较敏感的一连串数字，是在credit这一栏后面，赏金奖励，然后查阅二进制对照表解释，发现刚好前面是flag ，然后就解出来了
第五道题docx刚开始用wps显示隐藏文字，发现一串二进制，发现跟上题flag一样，提交了，结果发现并不是，然后提示是那个docx文件本质，上网查了一下，发现是zip压缩文件，然后把后缀改了，再解压缩出来，就找到flag了；
第六道题是关于流量分析的，首先去官网下了个wireshark，然后把那个文件用这个打开，根据提示，筛选http包，然后去网上搜了下相关的流量分析教程，在封包列表info中查看，有没有比较关键的东西，比如压缩文件、文本文件、音频文件、图片等，结果发现有一个 get / s3cret.png ，是个图片，而且还secret，判定应该就是这个，然后右键 follow - http stream 查看有没有flag，发现直接信息，然后就选择吧http传输的文件内容提取出来，选中刚才那个http包，file - export objects - http save all ，然后到文件夹里查看，果然有一个s3cret.png ,打开找到flag
